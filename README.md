## Key Results (Net Sharpe, 5 bps one-way TC)

### Full Period 2010–2026 — ATCClassifierScore, Weekly+20d horizon

| Universe | Gross Sharpe | **Net Sharpe** | Avg Turnover | Notes |
|---|---|---|---|---|
| SP500 | 1.371 | **1.230** | 99.9% | PIT membership via Wikipedia ✓ |
| SP1500 | 1.558 | **1.421** | 100.0% | SP400/SP600 PIT approx pre-2012/2019¹ |
| RU3K proxy | 1.411 | **1.342** | 100.0% | All US-listed ATC tickers, not true R3K² |

> ¹ SP1500 Pre-2020 Sharpe of 2.024 (from `summary_results_improved.csv`) is partly
> inflated by survivorship bias from the SP400/SP600 PIT approximation. The
> post-2020 period (Sharpe 0.71) uses more complete PIT data and is more reliable.
>
> ² RU3K proxy = ~7,372 US-listed ATC tickers vs true Russell 3000 ~3,000.
> Results are an upper bound vs genuine Russell 3000 membership.

> **Turnover note:** ~100% turnover per period is correct for this strategy design —
> the portfolio is fully reconstructed each week using that week's earnings events.
> Different companies report each week, so there is minimal position overlap across periods.
> An overlapping event-hold implementation would show lower effective turnover.

## Look-Ahead Bias Audit

See `look_ahead_audit_checklist.txt` (15 items, all pass).

Key methodology notes:
- `sentences_sector_z` uses `shift(1)` before expanding — current call not in its own z-score
- `sector_pct_rank` sorted by `entry_date` (trade date) not `DocDate`
- Rolled-entry forward returns use `actual_entry_date + h BDays` for the exit (not original date)
- Fluff/Filler columns kept in `signals.parquet` — placebo signal is non-zero (mean=0.028, std=0.021)
- INGESTDATEUTC verified as batch export timestamp (mean lag = 4,833 days); entry uses MOSTIMPORTANTDATEUTC only
